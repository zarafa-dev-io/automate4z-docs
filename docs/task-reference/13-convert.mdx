# `convert` — Conversion de formats

Convertit des données entre formats structurés. v1.0 : JSON, YAML, CSV. v1.x : XML, TSV.

```yaml
- uses: convert
  with:
    src: ./data/clients.csv
    from: csv
    to: json
    dst: ./data/clients.json
```

## Inputs

| Champ | Type | Requis | Description |
|-------|------|--------|-------------|
| `src` | string | l'un des deux | Chemin local du fichier source. |
| `content` | string ou object | l'un des deux | Contenu inline. |
| `from` | string | oui | `json`, `yaml`, `csv`. |
| `to` | string | oui | Idem `from`. |
| `dst` | string | non | Fichier de sortie. Si absent, dans `outputs.result`. |
| `options` | object | non | Options spécifiques au format (voir ci-dessous). |

### Options CSV

| Champ | Type | Défaut | Description |
|-------|------|--------|-------------|
| `csvDelimiter` | string | `,` | Séparateur. |
| `csvHeader` | boolean | `true` | Première ligne = en-tête. |
| `csvQuote` | string | `"` | Caractère de quoting. |

### Options JSON

| Champ | Type | Défaut |
|-------|------|--------|
| `jsonIndent` | integer | `2` — `0` = compact |

### Options YAML

| Champ | Type | Défaut |
|-------|------|--------|
| `yamlIndent` | integer | `2` |
| `yamlFlowStyle` | boolean | `false` |

### Options d'aplatissement

| Champ | Type | Défaut | Description |
|-------|------|--------|-------------|
| `flatten` | boolean | `false` | Aplatir les objets imbriqués pour CSV. |
| `flattenSeparator` | string | `.` | Séparateur (`user.name`). |

## Outputs

| Output | Type | Description |
|--------|------|-------------|
| `result` | any | Données converties si `dst` non fourni. |
| `rowCount` | integer | Pour CSV : nombre de lignes (hors header). |
| `bytesWritten` | integer | Si `dst` fourni. |

## Exemples

**CSV (issu d'un batch z/OS) vers JSON pour API**

```yaml
- id: read-export
  uses: fs
  with:
    action: read
    path: ./exports/clients.csv

- id: as-json
  uses: convert
  with:
    content: ${{ steps.read-export.outputs.content }}
    from: csv
    to: json
    options:
      csvHeader: true
      jsonIndent: 0

- uses: http
  with:
    method: POST
    url: https://api.corp/clients/import
    body: ${{ steps.as-json.outputs.result }}
```

**YAML manifeste vers JSON pour outil tiers**

```yaml
- uses: convert
  with:
    src: ./z-manifest.yaml
    from: yaml
    to: json
    dst: ./build/z-manifest.json
```

**JSON imbriqué vers CSV plat**

```yaml
- uses: convert
  with:
    content: ${{ steps.api.outputs.body }}
    from: json
    to: csv
    dst: ./report.csv
    options:
      flatten: true
      csvHeader: true
      csvDelimiter: ';'
```

## Erreurs courantes

- **CSV sans header en mode `csvHeader: true`** — mauvais parsing. Vérifier le fichier.
- **Encodage source** — un CSV en Windows-1252 lu en UTF-8 produit des caractères corrompus. Pré-convertir via `fs.read` avec `encoding:`.
