# automate4z

**automate4z** est un orchestrateur de workflows open-source qui efface la frontière entre exécution mainframe z/OS et exécution locale.

Distribué comme **un binaire unique `a4z`** (Linux / macOS / Windows × amd64 / arm64), sans runtime à installer côté utilisateur. Les workflows sont définis en YAML avec une syntaxe inspirée de GitHub Actions, adaptée au mainframe.

---

## Pourquoi automate4z ?

Les équipes COBOL en transition DevOps font face à un problème récurrent : leurs outils CI/CD modernes ne savent pas parler au mainframe. Chaque pipeline contient des scripts maison qui soumettent des jobs JCL, téléchargent des datasets et vérifient des return codes — le tout sans type safety ni observabilité.

automate4z résout ça avec un seul fichier YAML, un seul binaire, et une interface cohérente pour les deux mondes.

---

## Fonctionnalités clés

| Fonctionnalité | Description |
|----------------|-------------|
| **16 tâches built-in** | Shell, HTTP, filesystem, archives, notifications, requêtes JSON/YAML, templates, z/OS jobs et datasets |
| **Exécution DAG** | Dépendances entre steps résolues automatiquement ; steps indépendants en parallèle |
| **Expressions `$\{{ }}`** | Interpolation de contexte (inputs, outputs, env, runner) avec fonctions built-in |
| **Profils z/OSMF** | Connexion aux LPARs via profils Zowe ou variables d'env ; secrets jamais dans le YAML |
| **Lint + Plan** | Valide un workflow et affiche le plan d'exécution sans l'exécuter |
| **Binaire unique** | Pas de Node.js, Python ou Java requis — un seul exécutable cross-platform |

## Exemple minimal

```yaml
apiVersion: automate4z/v1
kind: Workflow
name: hello-world

inputs:
  name:
    type: string
    default: World

steps:
  - id: greet
    run: echo "Hello, ${{ inputs.name }}!"
```

```bash
a4z run hello.yml --input name=Alice
```
