# `git` — Local Git operations

Operations on a local Git repository. v1.0: **read-only** actions (`clone`, `checkout`, `status`, `log`). Write actions (`commit`, `tag`, `push`…) are coming in v1.x.

Implementation: wrapper around the local **`git` CLI** (not `go-git`), for 100% compatibility with the user's usual Git experience.

```yaml
- uses: git
  with:
    action: clone
    url: https://github.com/org/repo.git
    path: ./vendor/repo
```

## Common inputs

| Field | Type | Required |
|-------|------|----------|
| `action` | string | yes — `clone`, `checkout`, `status`, `log` |

### Action `clone`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `url` | string | yes | — | Repository URL. |
| `path` | string | yes | — | Destination directory. |
| `ref` | string | no | `HEAD` | Branch, tag, or SHA. |
| `depth` | integer | no | — | Shallow clone (`--depth N`). |
| `submodules` | boolean | no | `false` | Clone submodules recursively. |

### Action `checkout`

| Field | Type | Required |
|-------|------|----------|
| `path` | string | yes — repository directory |
| `ref` | string | yes — branch, tag, or SHA |
| `force` | boolean | no — default `false` |

### Action `status`

| Field | Type | Required |
|-------|------|----------|
| `path` | string | yes — repository directory |

### Action `log`

| Field | Type | Required | Default |
|-------|------|----------|---------|
| `path` | string | yes | — |
| `range` | string | no | `HEAD` — e.g. `v1.0..HEAD`, `--since=1week` |
| `count` | integer | no | `100` |
| `format` | string | no | Default structured format |

## Outputs

| Action | Outputs |
|--------|---------|
| `clone` | `path`, `commit` (SHA), `branch` |
| `checkout` | `commit`, `branch`, `ref` |
| `status` | `clean` (bool), `modified` (array), `untracked` (array), `staged` (array), `branch` |
| `log` | `commits` (array of `{ sha, author, email, date, subject, body }`), `count` |

## Examples

**Clone a specific tag**

```yaml
- uses: git
  with:
    action: clone
    url: https://github.com/example/cobol-common.git
    path: ./vendor/cobol-common
    ref: v2.1.0
    depth: 1
```

**Verify the repo is clean before a build**

```yaml
- id: status
  uses: git
  with:
    action: status
    path: .

- uses: assert
  with:
    condition: ${{ steps.status.outputs.clean == true }}
    message: 'Working tree must be clean before build'
```

**Generate a changelog**

```yaml
- id: changelog
  uses: git
  with:
    action: log
    path: .
    range: v${{ inputs.previousVersion }}..HEAD
    count: 1000

- uses: template
  with:
    content: |
      # Changelog ${{ inputs.version }}

      <<range .commits>>
      - <<.subject>> (<<slice .sha 0 7>>)
      <<end>>
    vars:
      commits: ${{ steps.changelog.outputs.commits }}
    dst: ./CHANGELOG.md
```

## Common errors

- **`fatal: Authentication failed`** — prefer a token via env var (`GH_TOKEN`) or the git credential helper.
- **`pathspec did not match any file`** on checkout — `ref` does not exist.
- **`status.clean` false when expected** — files generated by a previous build. Add them to `.gitignore`.
