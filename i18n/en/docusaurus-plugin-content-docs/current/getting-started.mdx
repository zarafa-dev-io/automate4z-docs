# Getting started

> **v1.1.0** — all local and z/OS tasks are stable. Features marked **(v1.x)** or **(v2)** are planned but not yet available.

```bash

```

Extract and add to your `PATH`:

```bash
tar -xzf automate4z_1.1.0_linux_amd64.tar.gz
mv a4z /usr/local/bin/
```

### Build from source

Requires Go 1.22 or later.

```bash
git clone https://github.com/zarafa-dev-io/automate4z.git
cd automate4z
go build -o a4z ./cmd/a4z
```

### Verify installation

```bash
a4z version
# a4z v1.1.0
```

## Your first workflow

Create a file `hello.yml`:

```yaml
apiVersion: automate4z/v1
kind: Workflow
name: hello-world

inputs:
  name:
    type: string
    default: World

steps:
  - id: greet
    run: echo "Hello, ${{ inputs.name }}!"

  - id: done
    run: echo "Workflow complete."
```

Run it:

```bash
a4z run hello.yml
a4z run hello.yml --input name=Alice
```

Expected output:

```
→ greet
  Hello, World!
✔ greet  0.12s

→ done
  Workflow complete.
✔ done  0.08s

Summary:
  ✔ greet  0.12s
  ✔ done   0.08s
```

## Workflow structure

A workflow is a single YAML file:

```yaml
apiVersion: automate4z/v1   # required
kind: Workflow               # required
name: my-workflow            # required

inputs:                      # optional
  module:
    type: string
    required: true
    default: COBPRG01

env:                         # optional — env vars injected in all steps
  TZ: Europe/Paris

defaults:                    # optional — default with values per task type
  shell:
    shell: bash

steps:                       # required, ≥ 1 step
  - id: build
    name: Compilation
    run: make build

outputs:                     # optional
  exitCode:
    value: ${{ steps.build.outputs.exitCode }}
```

### Step fields

| Field | Required | Description |
|-------|----------|-------------|
| `id` | yes | Unique identifier within the workflow |
| `name` | no | Label shown in logs (default: `id`) |
| `uses` | yes (unless `run`) | Built-in task name |
| `run` | yes (unless `uses`) | Shorthand for `uses: shell` |
| `with` | no | Task parameters |
| `needs` | no | Explicit dependencies |
| `if` | no | Execution condition |
| `env` | no | Step-specific env vars |
| `timeout` | no | Max duration (e.g. `30s`, `5m`, `1h`) |
| `continueOnError` | no | `true` to ignore step failure |

## Parallelism and DAG

### Default behavior: sequential

Without `needs`, steps run in **declaration order**.

### Explicit dependencies with `needs`

```yaml
steps:
  - id: fetch
    run: ./fetch-sources.sh

  - id: compile
    needs: [fetch]
    run: make compile

  - id: test
    needs: [fetch]       # parallel with compile
    run: make test

  - id: publish
    needs: [compile, test]   # waits for both
    run: make publish
```

`a4z plan` shows execution levels:

```
Level 1:  fetch
Level 2:  compile, test     ← parallel
Level 3:  publish
```

## Execution control

### Condition `if`

```yaml
# Run only on failure
- id: notify-failure
  if: ${{ failure() }}
  uses: http
  with:
    method: POST
    url: https://hooks.example.com/alert

# Always run (even after failure)
- id: cleanup
  if: ${{ always() }}
  run: rm -rf ./tmp

# Condition on an input
- id: publish-sonar
  if: ${{ inputs.publishSonar == true }}
  run: sonar-scanner
```

### Timeout

```yaml
- id: long-job
  timeout: 10m
  run: ./batch.sh
```

### `continueOnError`

```yaml
- id: optional-step
  continueOnError: true
  run: ./generate-report.sh
```

## Secrets

Secrets are passed via environment variables prefixed with `A4Z_SECRET_`. Their values are masked (`***`) in logs.

```bash
export A4Z_SECRET_API_TOKEN="ghp_xxxxx"
a4z run deploy.yml
```

In the workflow:

```yaml
- id: call-api
  uses: http
  with:
    url: https://api.example.com/deploy
    headers:
      Authorization: Bearer ${{ env.A4Z_SECRET_API_TOKEN }}
```

## Run artifacts

Each execution creates a directory under `.automate4z/runs/<runID>/`:

```
.automate4z/runs/
└── 01HZ4K3Q2RXXXXXXXXXXXXX/
    ├── workflow.yml
    ├── inputs.yml
    ├── run.json
    └── steps/
        ├── build/
        │   ├── stdout.log
        │   ├── stderr.log
        │   └── outputs.json
        └── test/
            ├── stdout.log
            ├── stderr.log
            └── outputs.json
```

:::tip
Add `.automate4z/` to your `.gitignore` to avoid committing run artifacts.
:::

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | Business failure (lint with errors, run with a failed step) |
| `2` | Internal error (file not found, invalid YAML, unexpected error) |

## Complete example — Local CI pipeline

```yaml
apiVersion: automate4z/v1
kind: Workflow
name: ci-local
description: Local CI pipeline — lint, build, tests, notification.

inputs:
  repo:
    type: string
    default: .
  notify:
    type: boolean
    default: false

steps:
  - id: vet
    uses: shell
    with:
      run: go vet ./...
      workdir: ${{ inputs.repo }}

  - id: lint
    uses: shell
    with:
      run: golangci-lint run ./...
      workdir: ${{ inputs.repo }}
      timeout: 3m

  - id: build
    needs: [vet, lint]
    run: go build -o ./bin/app ./cmd/app
    timeout: 5m

  - id: test
    needs: [vet, lint]
    uses: shell
    with:
      run: go test -race -count=1 ./...
      workdir: ${{ inputs.repo }}
      timeout: 10m

  - id: notify-ok
    needs: [build, test]
    if: ${{ success() && inputs.notify == true }}
    uses: http
    with:
      method: POST
      url: https://hooks.example.com/ci
      body:
        status: success
        workflow: ${{ workflow.name }}
        runId: ${{ workflow.runId }}

  - id: notify-fail
    needs: [build, test]
    if: ${{ failure() && inputs.notify == true }}
    uses: http
    with:
      method: POST
      url: https://hooks.example.com/ci
      body:
        status: failure
        workflow: ${{ workflow.name }}
```

```bash
a4z lint ci-local.yml
a4z plan ci-local.yml --input notify=true
a4z run  ci-local.yml --input notify=true --parallelism 2
```
